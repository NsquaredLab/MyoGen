Surface Electromyography (sEMG) Simulation.

This class provides a biophysical simulation framework for generating surface electromyography signals from the muscle. It implements a multi-layered cylindrical volume conductor model that accounts for anatomical accuracy, physiological variability, and measurement noise characteristics typical of clinical and research EMG recordings.

The simulation is built on established biophysical principles and validated anatomical data, making it suitable for algorithm development, method validation, and educational purposes in EMG signal processing and motor control research.

Scientific Foundation#

The simulation framework is based on:

  1. Volume Conductor Theory: Multi-layered cylindrical model representing muscle, fat, and skin tissues with anisotropic conductivity properties.

  2. Motor Unit Physiology: Realistic motor unit territories, fiber distributions, and action potential propagation based on anatomical and physiological measurements.

  3. Electrode Array Modeling: High-density surface electrode grids with configurable spatial arrangements and electrode properties.

  4. Signal Processing Pipeline: Complete workflow from single-fiber action potentials to composite surface EMG signals with noise characteristics.

Key Capabilities#

  • Multi-Position Electrode Arrays: Simulate EMG at multiple spatial locations

  • Motor Unit Action Potentials (MUAPs): Generate individual MUAP templates

  • Composite Surface EMG: Convolve MUAPs with spike trains for realistic EMG

  • Noise Modeling: Add measurement noise with specified signal-to-noise ratios

  • Computational Optimization: Efficient matrix reuse for large-scale simulations

  • Visualization Support: Built-in plotting capabilities for model validation

Typical Workflow#

  1. Create muscle model using myogen.simulator.Muscle

  2. Initialize SurfaceEMG with desired parameters

  3. Generate MUAP templates using simulate_muaps()

  4. Create motor neuron pool using myogen.simulator.MotorNeuronPool

  5. Generate surface EMG using simulate_surface_emg()

  6. Optionally add noise using add_noise()

Performance Considerations#

  • Memory usage scales as: n_positions × n_motor_units × grid_area × time_samples

  • Computational cost increases with: fiber count, electrode density, temporal resolution

  • For large simulations (>50 MUs, >64 electrodes), consider parameter optimization

  • GPU acceleration used automatically for convolution operations in surface EMG generation

Validation and Quality Assurance#

  • Parameters based on peer-reviewed anatomical and physiological studies

  • Built-in validation checks for parameter consistency

  • Extensive documentation with literature references

  • Type checking enforced via beartype decorators

Examples

Basic Usage:

>>> # Create muscle model
>>> muscle = myogen.simulator.Muscle(...)
>>>
>>> # Initialize surface EMG simulator
>>> surface_emg = SurfaceEMG(
...     muscle_model=muscle,
...     electrode_grid_dimensions__rows_cols=(8, 8),
...     MUs_to_simulate=[0, 1, 2, 3, 4]  # First 5 motor units
... )
>>>
>>> # Generate MUAP templates
>>> muaps = surface_emg.simulate_muaps(verbose=True)
>>>
>>> # Create motor neuron pool and generate surface EMG
>>> motor_pool = myogen.simulator.MotorNeuronPool(...)
>>> surface_emg_signals = surface_emg.simulate_surface_emg(motor_pool)
>>>
>>> # Add measurement noise
>>> noisy_emg = surface_emg.add_noise(snr_db=15)

Multi-Position Analysis:

>>> positions = [
...     (0, 0),                    # Center
...     (-5, np.deg2rad(45)),     # 5mm proximal, 45° rotation
...     (5, np.deg2rad(90)),      # 5mm distal, 90° rotation
... ]
>>> surface_emg = SurfaceEMG(
...     muscle_model=muscle,
...     electrode_grid_center_positions=positions,
...     MUs_to_simulate=list(range(20))  # First 20 MUs
... )
>>> muaps = surface_emg.simulate_muaps()
>>> print(f"Multi-position MUAPs shape: {muaps.shape}")  # (3, 20, 8, 8, 256)

Initialize the Surface EMG simulator with anatomical, physiological, and technical parameters.

This constructor sets up all parameters required for realistic surface EMG simulation, including muscle geometry, tissue properties, electrode configuration, and simulation resolution. All parameters are based on published anatomical and physiological studies to ensure biological realism.

param muscle_model:

Pre-computed muscle model containing motor unit territories, fiber distributions, and recruitment patterns. Must be created using myogen.simulator.Muscle class. The muscle model defines the spatial organization of motor units and their associated muscle fibers within the FDI muscle volume.

type muscle_model:

Muscle

param sampling_frequency__Hz:

Temporal sampling frequency for MUAP and EMG signals in Hertz. Higher frequencies provide better temporal resolution but increase computational cost and memory usage. Typical range: 1024-4096 Hz. - Clinical EMG: 1024-2048 Hz - Research applications: 2048-4096 Hz - High-resolution studies: >4096 Hz

type sampling_frequency__Hz:

float, default=2048.0

param mean_conduction_velocity__mm_s:

Mean conduction velocity of action potentials along muscle fibers in mm/s. Based on experimental measurements from Botelho et al. (2019). This parameter significantly affects MUAP duration and propagation patterns. Typical range for FDI muscle: 3.5-4.5 mm/s.

type mean_conduction_velocity__mm_s:

float, default=4.1

param sampling_points_in_t_and_z_domains:

Spatial and temporal discretization resolution for numerical integration. Controls the accuracy of the volume conductor calculations but significantly impacts computational cost (scales quadratically). Higher values provide better numerical accuracy at the expense of simulation time. - Fast simulations: 128-256 points - Standard accuracy: 256-512 points - High accuracy: 512-1024 points

type sampling_points_in_t_and_z_domains:

int, default=256

param sampling_points_in_theta_domain:

Angular discretization for cylindrical coordinate system in degrees. 180 points provides 2° angular resolution, which is sufficient for most surface EMG applications. Higher values provide better spatial resolution for circumferential electrode placement studies.

type sampling_points_in_theta_domain:

int, default=180

param MUs_to_simulate:

Indices of specific motor units to simulate. If None, simulates all motor units present in the muscle model. For computational efficiency, consider simulating subsets for initial analysis or algorithm development. Motor unit indices correspond to recruitment order (0 = first recruited).

type MUs_to_simulate:

list[int] or None, default=None

param mean_fiber_length__mm:

Mean muscle fiber length in millimeters based on anatomical measurements (Jacobson et al., 1992). This parameter affects MUAP duration and amplitude characteristics. Longer fibers produce longer-duration MUAPs with potentially higher amplitudes. Typical range for FDI: 28-36 mm.

type mean_fiber_length__mm:

float, default=32

param var_fiber_length__mm:

Standard deviation of fiber length distribution in millimeters. Introduces realistic biological variability in MUAP shapes within motor units. Higher values increase MUAP shape variability but maintain physiological realism. Typical range: 2-5 mm.

type var_fiber_length__mm:

float, default=3

param radius_bone__mm:

Inner bone radius for cylindrical volume conductor model in millimeters. For surface EMG applications, bone effects are typically minimal, so default value of 0 is appropriate. Non-zero values may be used for intramuscular EMG or detailed anatomical studies.

type radius_bone__mm:

float, default=0

param fat_thickness__mm:

Subcutaneous fat layer thickness in millimeters. This parameter significantly affects MUAP amplitude and spatial distribution due to fat’s low electrical conductivity acting as an insulator. Based on ultrasound measurements (Störchle et al., 2018). Typical range: 0.5-3 mm for hand muscles.

type fat_thickness__mm:

float, default=1

param skin_thickness__mm:

Skin layer thickness in millimeters. Generally less critical than fat thickness but affects high-frequency components of MUAPs and electrode-tissue coupling. Based on histological measurements (Brodar, 1960). Typical range: 0.5-2 mm.

type skin_thickness__mm:

float, default=1

param muscle_conductivity_radial__S_m:

Muscle tissue electrical conductivity in radial direction (perpendicular to fibers) in Siemens per meter. Muscle tissue exhibits anisotropic conductivity due to fiber orientation, with lower conductivity perpendicular to fibers. Value based on Botelho et al. (2019).

type muscle_conductivity_radial__S_m:

float, default=0.09

param muscle_conductivity_longitudinal__S_m:

Muscle tissue electrical conductivity in longitudinal direction (parallel to fibers) in Siemens per meter. Approximately 4-5x higher than radial conductivity due to fiber structure and orientation. This anisotropy significantly affects MUAP spatial patterns and propagation characteristics.

type muscle_conductivity_longitudinal__S_m:

float, default=0.4

param fat_conductivity__S_m:

Subcutaneous fat tissue electrical conductivity in Siemens per meter. Low conductivity makes fat act as an electrical insulator, spatially smoothing MUAP patterns and reducing amplitudes. Based on experimental measurements from tissue impedance studies.

type fat_conductivity__S_m:

float, default=0.041

param skin_conductivity__S_m:

Skin tissue electrical conductivity in Siemens per meter. Relatively high conductivity provides good electrical coupling between underlying tissues and surface electrodes. Value based on Roeleveld et al. (1997).

type skin_conductivity__S_m:

float, default=1

param electrode_grid_inclination_angle__deg:

Rotation angle of electrode grid relative to muscle fiber direction in degrees. - 0° = electrodes aligned parallel to muscle fibers - 90° = electrodes aligned perpendicular to muscle fibers This parameter affects MUAP propagation patterns and is important for studies of fiber direction estimation and conduction velocity analysis.

type electrode_grid_inclination_angle__deg:

float, default=0

param electrode_grid_dimensions__rows_cols:

Number of electrode rows and columns in the recording grid. Total number of electrodes = rows × columns. Common configurations: - (8, 8): 64 electrodes - standard high-density EMG - (16, 8): 128 electrodes - very high spatial resolution - (13, 5): 65 electrodes - clinical electrode arrays Higher densities provide better spatial resolution but increase data volume.

type electrode_grid_dimensions__rows_cols:

tuple[int, int], default=(8, 8)

param inter_electrode_distance__mm:

Distance between adjacent electrodes in millimeters. Affects spatial resolution and signal crosstalk between channels. Smaller distances provide higher spatial resolution but may increase correlated noise between adjacent channels. - High resolution: 2-3 mm - Standard: 4-5 mm - Low resolution: 8-10 mm

type inter_electrode_distance__mm:

float, default=4

param electrode_radius__mm:

Physical radius of individual circular electrodes in millimeters. Affects spatial averaging (larger electrodes average over more tissue volume) and signal-to-noise ratio (larger electrodes typically have lower impedance). Standard clinical surface electrodes: 0.5-2 mm radius.

type electrode_radius__mm:

float, default=0.75

param electrode_grid_center_positions:

List of electrode grid center positions in cylindrical coordinates. Each position is specified as (z_position_mm, angle_radians) where: - z_position_mm: position along muscle fiber direction (+ = distal, - = proximal) - angle_radians: circumferential position around muscle (0 = anterior)

If None, uses single position at anatomical muscle belly center computed from the center of mass of all motor unit territories.

Multiple positions enable spatial analysis of EMG characteristics across different muscle regions. Example positions: - [(0, 0)]: Single center position - [(0, 0), (-10, 0), (10, 0)]: Proximal-center-distal sequence - [(0, 0), (0, π/2), (0, π)]: Circumferential positions

type electrode_grid_center_positions:

list[tuple[float|int, float|int]] or None, default=None

  • snr_db (float) – Target Signal-to-Noise Ratio in decibels (dB). Higher values mean less noise. Typical values for surface EMG: - High quality: 20-30 dB - Medium quality: 10-20 dB - Low quality: 0-10 dB - Very noisy: < 0 dB

  • noise_type (str, default="gaussian") – Type of noise to add. Currently supports: - “gaussian”: Additive white Gaussian noise (most common for EMG)

Returns:

noisy_surface_emg – Surface EMG signals with added noise, same shape as self.surface_emg__tensor. The original signal is preserved in self.surface_emg__tensor.

Return type:

np.ndarray

Raises:

ValueError – If surface EMG has not been simulated yet (call simulate_surface_emg first) If noise_type is not supported

Notes

SNR Calculation: SNR_dB = 10 * log10(P_signal / P_noise)

Where: - P_signal = mean power of the signal across all channels - P_noise = power of the additive noise

The noise power is calculated as: P_noise = P_signal / (10^(SNR_dB/10))

Noise Characteristics: - Gaussian noise: Zero-mean with variance calculated to achieve target SNR - Noise is added independently to each electrode channel - Noise power is calculated based on the RMS power of the signal

Examples

>>> # Simulate surface EMG first
>>> surface_emg = emg_simulator.simulate_surface_emg(motor_neuron_pool)
>>>
>>> # Add moderate noise (15 dB SNR)
>>> noisy_emg = emg_simulator.add_noise(snr_db=15)
>>>
>>> # Add high noise (5 dB SNR)
>>> very_noisy_emg = emg_simulator.add_noise(snr_db=5)
>>>
>>> print(f"Original EMG shape: {surface_emg.shape}")
>>> print(f"Noisy EMG shape: {noisy_emg.shape}")
simulate_muaps(show_plots=False, verbose=True)[source]#

Simulate surface EMG Motor Unit Action Potentials (MUAPs) for the FDI muscle.

This function implements a comprehensive biophysical model that simulates surface EMG signals by computing the electrical activity of individual muscle fibers within motor units and their propagation through a multi-layered volume conductor (muscle, fat, skin).

The simulation is optimized for multiple electrode positions by reusing computationally expensive matrix calculations across positions, significantly reducing simulation time for spatial EMG analysis.

Scientific Basis#

The simulation implements the forward problem of EMG using:

1. Cylindrical Volume Conductor Model: The limb is modeled as a multi-layered cylinder with realistic tissue conductivities accounting for anisotropy in muscle tissue.

2. Action Potential Propagation: Intracellular action potentials propagate along muscle fibers with physiologically realistic conduction velocities and fiber lengths.

3. Spatial Integration: Surface potentials are computed by spatially integrating contributions from all active muscle fibers within each motor unit territory.

Optimization Strategy#

For multiple electrode positions, the function employs matrix reuse optimization: - Computes position-independent matrices (A_matrix, B_incomplete) once per fiber - Reuses these matrices across all electrode positions - Reduces computational complexity from O(n_positions × n_fibers × matrix_ops) to O(n_fibers × matrix_ops + n_positions × vector_ops)

simulate_surface_emg(motor_neuron_pool)[source]#

Generate realistic surface EMG signals by convolving MUAP templates with motor neuron spike trains.

This method implements the final step of surface EMG simulation by combining the previously computed Motor Unit Action Potential (MUAP) templates with physiologically realistic motor neuron firing patterns. The process involves temporal resampling to match simulation timesteps, convolution of spike trains with MUAP shapes, and summation across all active motor units for each electrode channel.

The simulation can optionally use GPU acceleration (via CuPy) for efficient parallel processing of convolution operations across multiple electrode positions, motor units, and time points. If CuPy is not available, the simulation automatically falls back to CPU computation with NumPy.

Scientific Background#

Surface EMG signals arise from the temporal and spatial summation of individual motor unit action potentials (MUAPs) as they propagate through tissue and are detected by surface electrodes. The mathematical relationship is:

EMG(t) = Σᵢ Σⱼ MUAP_i(t - tᵢⱼ)

Where: - i indexes motor units - j indexes spike times for motor unit i - MUAP_i(t) is the MUAP template for motor unit i - tᵢⱼ is the j-th spike time of motor unit i

This summation is efficiently computed using convolution: EMG(t) = Σᵢ spike_train_i(t) ⊛ MUAP_i(t)

Processing Pipeline#

  1. Temporal Resampling: Adjust MUAP sampling rate to match motor neuron simulation timestep

  2. Motor Unit Selection: Identify active motor units from intersection of simulated MUs and spike trains

  3. Convolution: Convolve each spike train with corresponding MUAP template

  4. Spatial Summation: Sum contributions across all active motor units for each electrode

  5. Multi-Position Processing: Repeat for all electrode grid positions

GPU Acceleration#

The method automatically uses GPU processing (if CuPy is available) for: - Large-scale convolution operations across multiple dimensions - Parallel processing of electrode positions and motor unit combinations - Efficient memory management for large EMG datasets

If CuPy is not installed or GPU is unavailable, the method automatically falls back to CPU-based computation using NumPy with equivalent functionality.

param motor_neuron_pool:

Motor neuron pool containing spike trains and simulation parameters. Must contain: - spike_trains: Binary spike train matrix [n_pools, n_motor_units, n_time_samples] - timestep__ms: Simulation timestep in milliseconds - active_neuron_indices: List of active motor unit indices for each pool

The motor neuron pool defines the temporal firing patterns that will be convolved with the MUAP templates to generate surface EMG signals.

type motor_neuron_pool:

MotorNeuronPool

returns:

surface_emg__tensor – Multi-dimensional surface EMG signals with shape: (n_positions, n_pools, n_electrode_rows, n_electrode_cols, n_time_samples)

Dimensions: - n_positions: Number of electrode grid positions - n_pools: Number of motor neuron pools (different simulation conditions) - n_electrode_rows: Number of electrode rows in grid - n_electrode_cols: Number of electrode columns in grid - n_time_samples: Number of temporal samples in EMG signal

Units: Volts (typically in microvolts range for surface EMG)

The tensor can be indexed as: surface_emg[position_idx, pool_idx, row, col, :] -> EMG time series for specific electrode

Notes

Computational Complexity: - Time complexity: O(n_positions × n_pools × n_MUs × n_electrodes × n_time_samples) - Space complexity: O(n_positions × n_pools × n_electrodes × n_time_samples) - GPU acceleration (if available) provides 10-50x speedup for large simulations

Signal Characteristics: - Amplitude range: typically 10-1000 μV for surface EMG - Frequency content: 10-500 Hz (depending on muscle and electrode configuration) - Temporal patterns reflect motor unit recruitment and firing rate modulation - Spatial patterns depend on motor unit territories and electrode positioning

Validation Recommendations: - Verify EMG amplitude scaling with muscle activation level - Check frequency content matches physiological expectations - Ensure spatial patterns are consistent with motor unit territories - Compare multi-channel correlations with experimental data
